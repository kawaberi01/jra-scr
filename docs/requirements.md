# JRAレース情報取得基盤 要件定義書

## 目的

- JRA公式サイトを主な情報源として、レース情報を安定して取得できる基盤を作る
- ローカルPC上で HTTP API として動作させ、将来的に MCP/スキルから利用できる形にする
- 特に、実行時点での各買い方のオッズを確実に取得できる構成を優先する

## 対象

- 開催日単位のレース一覧
- レース単位の出馬表
- レース単位のオッズ
- レース単位の結果

## 方針

- 既存OSSの設計思想を吸収し、JRA向け実装として再構成する
- 取得層、抽出層、正規化層、API層を分離する
- HTTP取得を主経路にし、必要時のみブラウザ自動化をフォールバックにする
- セレクタや抽出ルールはコードから切り離し、JSONで定義する

## 優先度

1. race_id 解決
2. 出馬表取得
3. オッズ取得
4. 結果取得
5. HTTP API 化
6. MCP ラッパー
