# 現在地と次の一手

## 現在地

ここまでで、設計書と実装計画に対して次は完了済みである。

- JRA navigation 層
- meeting 一覧 API
- 開催地座標からの出馬表取得
- 開催地座標からのオッズ取得
  - `win`
  - `quinella`
  - `exacta`
  - `wide`
  - `trio`
  - `trifecta`
- 開催地座標からの結果・払戻取得
- 過去結果バッチ
  - JSONL 永続化
  - `race_id` 単位 skip
  - 固定回数 retry
- MCP HTTP 入口 (`/mcp`)
- テスト整備

## 現在のギャップ

- formal docs は今回作成したが、今後は feature ごとに同期運用へ移す必要がある
- batch 保存先は現状 JSONL のみで、DB やオブジェクトストレージ abstraction までは未対応
- MCP ではまだ自然言語から `course` / `race_no` / `bet_type` を解釈していない

## 次の一手

最も自然なのは次である。

1. MCP 入力正規化
- `中山` -> `nakayama`
- `11R` -> `race_no=11`
- `馬連` -> `quinella`

2. batch 保存先 abstraction の拡張
- JSONL 以外の backend
- 運用時のローテーションや保存先切替

3. formal docs / 利用ガイドの feature 単位同期運用
- 変更ごとに docs/jra を追随させる流れの定着

FastAPI API の MCP 入口までは追加できたため、次は MCP で実用になる入力正規化を足す段階に入る。
