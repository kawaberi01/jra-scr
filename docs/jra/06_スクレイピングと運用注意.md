# スクレイピングと運用注意

## 前提

このプロジェクトは JRA 公式サイトの HTML 導線に依存する。
そのため、サイト構造変更の影響を受ける。

## 注意点

- `JRADB/access*.html` は単純 GET だけで完結しない
- `doAction(..., cname)` に相当する POST / GET を使い分ける必要がある
- `cname` は内部導線用であり、外部 API には露出させない
- fixture が regressions の基準になる

## 文字コード

- fixture は `utf-8` 優先
- 読めない場合は `shift_jis` にフォールバックする

## キャッシュ

- odds は短 TTL
- result は長め TTL
- `refresh=true` で odds 再取得を優先できる

## 現在のリスク

- 未対応券種追加時は odds navigation と parser を増やす必要がある
- batch は保存未実装なので、再収集コストが残る
- live fetch の実戦運用では rate や失敗 retry をまだ詰めていない
