## 2026-07-01 正式 CLI 化

### 実装済み

- `.workstate` の補助スクリプトに依存しない正式 CLI として `backfill-analysis-runners` と `verify-analysis-joins` を追加。
- 本体責務は `src/jra_srb/analysis_maintenance.py` に切り出し、CLI は引数解釈と exit code 判定を担当。
- `backfill-analysis-runners` は既存 `races` を正として対象 race を抽出し、`--only-missing` 指定時は `runners` 0 件の race だけを処理。
- `--dry-run` は対象抽出と件数表示のみを行い、`collection_runs` / `runners` へ書き込まない。
- `JraService.get_race_card_by_number()` で取得した `RaceCard` を `AnalysisSQLiteStore.write_card()` で保存。
- run は `collection_runs`、race 単位の失敗は `collection_errors` に記録し、失敗があっても可能な限り継続。
- `write_card()` の upsert により、再実行しても `runners` は重複しない。
- `verify-analysis-joins` は races 基準の件数、join 件数、runners 欠落 race 数、latest run errors、sample runners を出力し、0 件または join 不成立時に非 0 exit。
