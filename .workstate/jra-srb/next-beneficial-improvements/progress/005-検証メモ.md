# 検証メモ

## 2026-06-28

実行:

```bash
rtk uv run --extra dev pytest -q
```

結果:

- 失敗。
- `.venv` が壊れており、Python executable が存在しないため `uv` が実行できなかった。

再実行:

```bash
rtk .venv-win\Scripts\python.exe -m pytest -q
```

結果:

- 57 passed
- 2 failed
- 失敗は `tests/test_auto_progress.py` の外部スキルスクリプト解決が `D:\home\...` になっている環境依存エラー。
- 今回変更した JRA API / batch / CLI / cache の失敗ではない。

変更対象確認:

```bash
rtk .venv-win\Scripts\python.exe -m pytest -q -k "not auto_progress"
```

結果:

- 57 passed
- 2 deselected
- 1 warning

uv での再確認:

```bash
rtk powershell.exe -NoProfile -Command "`$env:UV_PROJECT_ENVIRONMENT='.venv-win'; uv run --extra dev pytest -q -k 'not auto_progress'"
```

結果:

- 57 passed
- 2 deselected
- 1 warning

補足:

- Windows 側では `.venv` が Linux 形式で `Scripts/python.exe` を持たないため、`uv run` はそのままだと失敗する。
- `.venv-win` は有効な Windows 仮想環境なので、`UV_PROJECT_ENVIRONMENT=.venv-win` を指定すれば `uv run` で実行できる。
