# 進捗サマリ

## 2026-06-28

- `030-api-quality-next-tasks実装指示書.md` に従い、P0 の Storage backend 設定統一と Upstream アクセス制御を実装対象に固定した。
- `JRA_SRB_RESULTS_STORAGE=jsonl|sqlite` を追加し、API 側でも JSONL / SQLite の保存済み結果 backend を選べるようにした。
- `JRA_SRB_RESULTS_STORAGE` の不正値は標準エラー形式の `bad_request` で返すようにした。
- `HttpProvider` に `max_concurrency` と `min_interval_seconds` を追加し、upstream への同時実行数と最小 request 間隔を制御できるようにした。
- `build_service()` で `JRA_SRB_UPSTREAM_MAX_CONCURRENCY` と `JRA_SRB_UPSTREAM_MIN_INTERVAL_SECONDS` から provider を構築するようにした。
- README と `docs/jra/05_API仕様.md` に追加環境変数を追記した。
