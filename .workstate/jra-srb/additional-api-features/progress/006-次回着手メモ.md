# 006-次回着手メモ

## 次回候補

1. Job registry の永続化
   - SQLite に job record を保存し、API サーバー再起動後も状態を参照できるようにする。
2. Job cancel / retry API
   - `DELETE /jobs/result-collections/{job_id}` または `POST /jobs/result-collections/{job_id}/retry` を検討する。
3. job 同時実行数制限
   - `JRA_SRB_JOB_MAX_CONCURRENCY` を追加し、収集 job の並列実行を制御する。
4. API key / CORS
   - 外部公開する場合の境界として実装する。
